Backend Error Test

Trigger real server-side errors to test the Orchestrator Agent auto-fix pipeline. Errors go through: ErrorLogger → Gemini Analysis → Orchestrator → Claude Code → PR.

Configuration

Select which organization and project to log errors against. The orchestrator will only auto-trigger if the project has AutoFixConfig enabled.

How Backend Errors Flow

POST /api/test-errors/* → ErrorLogger.logError() → PostgreSQL INSERT ErrorLog │ ├──→ Winston logs to file (immediate) ├──→ Gemini AI Analysis (async) → UPDATE ErrorLog.aiAnalysis ├──→ Email notification (async, if configured) │ └──→ PG NOTIFY 'error_analyzed' → Orchestrator DB Listener │ ├──→ AutoTriggerService.evaluate() (dedup, severity, cooldown) ├──→ WorkerPool.submitAnalysis() (max 5 workers) ├──→ ClaudeCodeRunner (focused prompt + cached context) ├──→ Status: AWAITING_APPROVAL │ └──→ Admin approves → PG NOTIFY 'pipeline_status' │ ├──→ Git worktree → Claude Code fix → Tests └──→ Commit → Push → GitHub PR Created

Trigger Backend Errors
