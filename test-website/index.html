CRM of Techview — Integration Test

This page tests all CRM features: error logging, AI chat widget, contact tracking, ticket creation, and the auto-fix pipeline.

1 Error Logging

Click buttons to trigger different error types. Each error is auto-captured by the SDK and sent to your CRM → Error Logs. Gemini AI will analyze ERROR/FATAL level errors.

Check in CRM: Admin → Error Logs — you'll see each error with level, source, stack trace
Auto-analyzed: ERROR and FATAL errors get Gemini AI root cause analysis
Email alert: If SMTP is configured, admins get an email with AI analysis

2 AI Chat Widget

The chat bubble in the bottom-right corner is your embeddable chatbot. It uses the chatbot configuration from your CRM project settings.

Try: Click the blue bubble → type "Hello" → AI responds using your CRM chatbot config
Ticket tab: Switch to "Create Ticket" in the widget → submit → check CRM → Tickets
Configure: CRM → Admin → Chatbot → change bot name, welcome message, system prompt

3 Contact Identification

When a user logs into your app, call identify() to create/update them as a contact in the CRM. This also links their chat sessions and tickets.

Check in CRM: CRM → Contacts — new contact appears with source "sdk-web"
Widget link: After identify, chat sessions are linked to the contact

4 Event Tracking

Track any custom event from your app — purchases, signups, button clicks, feature usage.

Check in CRM: These events are stored in SdkEvent table, linked to the API key

5 Programmatic Ticket Creation

Create support tickets from code — useful for automated bug reports, user feedback forms, or error escalation.

Check in CRM: Support → Tickets — ticket appears with "Submitted via SDK" note
AI analysis: The ticket gets auto-analyzed by Gemini for issue type and priority

6 Auto-Fix Pipeline (Claude Code)

After errors are logged, admins can trigger the auto-fix pipeline. Claude Code CLI analyzes the codebase and proposes a fix.

How to test locally:

Step 1: Go to CRM → Admin → Auto-Fix → VPS Agents tab → Register Agent
    Name: Local Test, Host: localhost, Path: your project path

Step 2: Copy the Agent Key from the setup instructions shown

Step 3: Open a new terminal and run:
cd "D:\Onkar\Office work\india\ai-support-saas\vps-agent" set CRM_URL=http://localhost:3001 set AGENT_KEY=vps_YOUR_KEY_HERE set PROJECT_PATH=D:\Onkar\Office work\india\ai-support-saas\server node agent.js
Step 4: Go to CRM → Error Logs → expand any error → click Auto-Fix with Claude Code

Step 5: Watch the agent terminal — Claude Code will analyze the code

Step 6: Go to CRM → Auto-Fix → see pipeline progress → Approve & Deploy
Pipeline stages: Detected → Analyzing → Fix Proposed → Awaiting Approval → Approved → Fixing → Committed → Deploying → Deployed
No API cost: Uses Claude Code CLI from your Max plan subscription
